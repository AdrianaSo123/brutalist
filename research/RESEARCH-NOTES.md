## Historical Timeline Reference

### 1950s - Architectural Origins
- **1952-1954**: Le Corbusier's Unité d'Habitation in Marseille
- **1953**: Term "Brutalism" coined by Alison & Peter Smithson
- **Philosophy**: Béton brut (raw concrete), truth to materials

### 1960s-1970s - Peak Brutalism
- Global expansion of Brutalist architecture
- Government buildings, universities, housing projects
- 1968: Barbican Centre construction begins
- Symbol of modernist progress and social democracy

### 1980s-2000s - Decline Era
- Hundreds of Brutalist buildings demolished
- Post-modernism rejection
- 1995: Craigslist launches (accidental Brutalism)
- Style dismissed as architectural mistake

### 2010s - Digital Revival
- 2012-2014: Reaction against over-polished interfaces
- 2014: Brutalist architecture renaissance
- 2016: BrutalistWebsites.com launches
- 2016: Washington Post election coverage uses Brutalist design

### 2020s - Mainstream Adoption
- High-fashion brands (Balenciaga) embrace Brutalist web design
- Integration into design systems
- Ongoing counter-aesthetic to over-designed interfaces

## Core Design Principles Extracted

### 1. Truth to Materials
- **Architectural**: Show concrete, don't hide structure
- **Digital**: Show grid, expose structure, make borders visible

### 2. Honest Function
- **Architectural**: Form follows function, no decoration for decoration's sake
- **Digital**: Every element has purpose, remove non-essentials

### 3. Anti-Corporate
- **Architectural**: Reject ornamental tradition, challenge aesthetics
- **Digital**: Deliberately rough, reject sleek interfaces

### 4. Raw Aesthetics
- **Architectural**: Exposed concrete, visible pipes, unfinished surfaces
- **Digital**: System fonts, thick borders, hard shadows, sharp corners

## Design Specifications Applied

### Color Palette
- Black: #000000 (dominant)
- White: #FFFFFF (contrast)
- Yellow: #FFFF00 (accent - bright, unrefined)
- No gradients, no subtle variations
- High contrast always

### Typography
- **Fonts**: Arial, Helvetica, Courier (system fonts only)
- **Headings**: ALL CAPS, 48-96px
- **Body**: 16-18px
- **Line-height**: 1.2-1.4 (tight)
- **Approach**: Extreme size contrast

### Layout & Borders
- **Border thickness**: 4-5px
- **Border color**: #000000
- **Corners**: 0px border-radius (90° angles only)
- **Grid**: Visible and exposed
- **Structure**: Boxed, compartmentalized sections

### Shadows
- **Type**: Hard, offset box-shadows
- **Values**: 8px 8px 0 #000000 (large), 6px 6px 0 (medium)
- **No blur**: First value is x-offset, second is y-offset, third is 0 (no blur)
- **Purpose**: Create depth without subtlety

### Spacing
- **Philosophy**: Tight and packed
- **Padding**: Minimal (16-32px typical)
- **Approach**: Intentional cramping creates tension
- **Contrast**: React against generous Swiss white space

### Interactions
- **Transitions**: none
- **Hovers**: Instant state changes
- **Animations**: None (smoothness = corporate)
- **Button effects**: Shadow shift + transform

## Common Mistakes Documented

### 1. Making It Too Pretty
- **Wrong**: Subtle shadows, rounded corners, smooth gradients
- **Right**: Hard shadows, 90° angles, flat colors
- **Why**: Brutalism rejects refinement

### 2. Using Too Many Colors
- **Wrong**: Rainbow palette, pastels, muted tones
- **Right**: Black, white, 1-2 bright accents
- **Why**: Brutalism is stark and high-contrast

### 3. Generous White Space
- **Wrong**: Lots of padding, airy layouts
- **Right**: Tight, packed, cramped
- **Why**: Swiss design is generous; Brutalism reacts against it

### 4. Smooth Animations
- **Wrong**: Fade transitions, ease curves
- **Right**: Instant state changes, hard snaps
- **Why**: Smoothness = polish = corporate

### 5. Hiding Structure
- **Wrong**: Seamless sections, invisible grids
- **Right**: Visible borders, exposed grid
- **Why**: Brutalism shows structure honestly

## Authenticity Criteria

Based on style guide, authentic Neo-Brutalism requires:

**Visual Style**:
- ✓ Stark black and white dominate
- ✓ Bright accent colors sparingly
- ✓ No subtle colors or gradients
- ✓ High contrast everywhere

**Typography**:
- ✓ System fonts only
- ✓ ALL CAPS headings
- ✓ Tight line-height
- ✓ Large size contrast

**Layout**:
- ✓ Thick borders (3-5px)
- ✓ No rounded corners
- ✓ Grid visible or obvious
- ✓ Boxed sections

**Spacing**:
- ✓ Tight and packed
- ✓ Minimal padding
- ✓ Intentional crowding

**Components**:
- ✓ Thick borders on buttons
- ✓ Hard shadows
- ✓ No smooth transitions
- ✓ Raw, unrefined appearance

**Overall Feel**:
- ✓ Deliberately rough (not accidentally amateur)
- ✓ Anti-corporate
- ✓ Honest about structure
- ✓ Functional and minimal

## Comparative Analysis: Neo-Brutalism vs Swiss Design

| Aspect | Swiss Design | Neo-Brutalism |
|--------|--------------|---------------|
| Grid | Invisible, mathematical | Visible, exposed |
| Spacing | Generous white space | Tight, packed |
| Typography | Refined hierarchy | Abrupt contrast |
| Colors | Minimal, refined | Stark black/white + bright |
| Borders | Thin or absent | Thick, black, dominant |
| Shadows | None or subtle | Hard, offset |
| Corners | Clean, subtle radius | Sharp 90° angles |
| Philosophy | Perfection, clarity | Honesty, rawness |
| Feel | Professional, elegant | Rough, rebellious |

## Implementation Notes

### Responsive Strategy
- Maintain thick borders on mobile (scale from 5px to 4px)
- Keep sharp corners (no border-radius even for touch)
- Reduce shadow offset (12px → 6px) but keep hard
- Stack layouts but preserve boxed structure
- Scale typography but maintain ALL-CAPS and contrast
- Tight spacing reduces but doesn't become generous

### Technical Constraints
- No JavaScript (honest, minimal)
- No frameworks (anti-bloat)
- System fonts only (no web fonts)
- Minimal dependencies
- Static HTML/CSS

### Content Strategy
- Short paragraphs for readability despite tight spacing
- Clear hierarchies through size, not subtlety
- Educational content balanced with brutalist aesthetic
- Historical accuracy with accessible language
